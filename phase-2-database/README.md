# Phase 2 — Database

**Thời gian**: 5–6 tuần | **Mục tiêu**: Thiết kế và tối ưu database như một pro

## Checklist

### SQL & PostgreSQL
- [ ] Relational model: table, row, column, PK, FK
- [ ] CRUD với SQL thuần
- [ ] JOIN: INNER, LEFT, RIGHT, FULL
- [ ] Index: tại sao cần, khi nào dùng, trade-off
- [ ] Transaction: ACID, BEGIN/COMMIT/ROLLBACK
- [ ] N+1 query problem
- [ ] Database normalization: 1NF, 2NF, 3NF
- [ ] Connection pooling (pg-pool)
- [ ] EXPLAIN / EXPLAIN ANALYZE

### ORM — Prisma
- [ ] Schema definition
- [ ] Migrations
- [ ] Relations (1-1, 1-N, N-N)
- [ ] Query optimization (include, select)
- [ ] Raw query

### NoSQL — MongoDB
- [ ] Document model vs Relational model
- [ ] CRUD
- [ ] Schema design: embedding vs referencing
- [ ] Mongoose
- [ ] Aggregation pipeline

### Redis — Caching
- [ ] Redis là gì — key-value, in-memory
- [ ] Cache-aside, Write-through, Write-behind
- [ ] Cache invalidation strategies
- [ ] TTL
- [ ] Redis pub/sub
- [ ] Session storage với Redis
